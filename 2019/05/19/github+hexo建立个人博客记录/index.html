github+hexo建立个人博客记录

一.准备：

1.git账号与gitpage仓库：

去github注册一个免费账号，这个不多说，不会的话网上搜一下教程，很多，你需要记录你的邮箱与用户名

新建一个仓库


图片示例1


这里库的名字必须与用户名一致，以表明这是一个账号唯一的gitpage

2.本机环境准备：

node.js:

下载地址：https://nodejs.org/en/

下载node.js环境，是为了建立hexo的运行环境

安装完成后，在windows cmd中输入node -v，可以得出版本号即可

git:

下载地址：https://git-scm.com/downloads

下载本地git，为本地项目提交github，以快速更新gitpage

安装完成后，打开gitbash，不知道怎样打开gitbash的话，在桌面右键，即可出现选项git bash here，

输入ssh-keygen -t rsa, 接着回车三下

出现一个奇怪的图形后，可以在C盘的用户下的特定用户文件夹中的.ssh文件夹下找到id_rsa.pub文件

如果找不到，在C盘的用户目录下，搜索id_rsa.pub，用记事本打开id_rsa.pub，

复制其中的全部内容，添加到GitHub上，


图片示例2


这样本地的id_rsa密钥就可以和GitHub上的id_rsa.pub公钥进行配对，授权成功

SSH key添加之后，就可以在本机git bash中进行测试，输入ssh -T git@github.com进行测试，

hexo:

由于npm上的hexo资源在国外，访问不稳定，所以这里先安装淘宝源，就是npm在国内的镜像，在cmd中输入，npm install -g cnpm -registry=https://registry.npm.taobao.org

重开cmd窗口，输入cnpm -v检测cnpm是否安装成功，

在cmd中输入cnpm install hexo-cli -g,这条是我猜的，没测试过，如果不行的话，输入npm install hexo-cli -g

然后在一个你想要的位置，创建一个空文件夹，点进去这个文件夹(我是D盘的hexo文件夹，所以下文用hexo文件夹指代这个你建立的文件夹)，右键 git bash here，输入hexo init，然后cnpm install hexo server

接着分别执行hexo g，hexo s，然后浏览器输入http://localhost:4000，应该可以看到hexo界面，说明前面hexo安装没啥大问题。

二.选择模板

现在去下载一个你喜欢的hexo模板，下载地址：https://hexo.io/themes/

点击图片即为该模板的效果预览，点击图片下面的文字标题，即为该模板的源码仓库，

我使用的是这个： https://github.com/Sariay/hexo-theme-Annie 预览地址在 https://sariay.github.io/

下载模板源码，可以在页面https://github.com/Sariay/hexo-theme-Annie中download zip，解压后放入那个你建的hexo文件夹的themes目录下，也可以在themes目录中git clone https://github.com/Sariay/hexo-theme-Annie

三.配置你的网站

然后，修改你建的hexo文件夹中的_config.yml

先打开配置文档_config.yml，对它做如下修改，repository后面的内容是 git@gitbub.com:username/库地址 的形式

注意：type、repository、branch冒号的后面都有一个空格


图片示例3


再修改_config.yml中的theme配置项，该值名称需要与你hexo/themes目录下的那个主题的文件夹名称相同


图片示例4


四.部署你的个人博客网站

在git bash中输入cnpm install hexo-deployer-git –save

然后输入 git config –global user.name “nx90”，把引号中的nx90换成你git的用户名

输入git config –global user.email “2321296607@qq.com“ 把引号中的邮箱名换为你github账号的邮箱，

输入hexo g，根据当前源码生成页面代码，

输入hexo d，部署到你的gitpage上，第一次输入hexo d可能会弹出一个登录框，你正常登录就好，

然后每次你对你的个人博客做修改的时候，都执行hexo g， hexo d

现在访问https://你的github用户名.github.io，即可看到你部署的个人博客了

五.定制化

后面的一些定制化的内容，则要根据每个主题的文档来自己定制，就自己慢慢根据需求去探索了，一般基础文档在主题项目的read.me中，我使用的Annie主题的进阶使用说明地址为https://sariay.github.io/2018/08/27/Annie%E4%B8%BB%E9%A2%98%E4%BD%BF%E7%94%A8%E8%AF%B4%E6%98%8E/

更多的功能如果有一定的前端能力也可以自己开发

最基础的写博客，是在hexo目录下的source/_posts目录中，放入你写好的markdown文件，也就是你的博客需要是markdown格式的，markdown格式文件在你还不了解markdown语法的情况下，最好使用有道云笔记或者其他的markdown编辑器，帮助你写md文件，
